# List Engines

List all engines available to your organization/user.

## Header Parameters

- Organization: string
-- Example: org-123456
*Allows for requests to be scoped to an organization other than the user's default. If not provided, the user's default organization will be used.*
- Stability-Client-ID: string
-- Example: my-great-plugin
*Used to identify the source of requests, such as the client application or sub-organization. Optional, but recommended for organizational clarity.*
- Stability-Client-Version: string
-- Example: 1.2.1
*Used to identify the version of the application or service making the requests. Optional, but recommended for organizational clarity.*

## Request Sample (cURL)

```bash
if [ -z "$STABILITY_API_KEY" ]; then
    echo "STABILITY_API_KEY environment variable is not set"
    exit 1
fi

BASE_URL=${API_HOST:-https://api.stability.ai}
URL="$BASE_URL/v1/engines/list"

curl -f -sS "$URL" \
  -H 'Accept: application/json' \
  -H "Authorization: Bearer $STABILITY_API_KEY"
```

## Responses

### 200 OK response

#### Response Schema: application/json

Array of:

- id (required): string
*Unique identifier for the engine.*
- name (required): string
*Name of the engine.*
- description (required): string
- type (required): string
-- Enum: AUDIO CLASSIFICATION PICTURE STORAGE TEXT VIDEO
*The type of content this engine produces.*

### 401 unauthorized: API key missing or invalid

#### Response Schema: application/json

- name (required): string
*The short-name of this class of errors e.g. bad_request.*
- id (required): string
*A unique identifier for this particular occurrence of the problem.*
- message (required): string
*A human-readable explanation specific to this occurrence of the problem.*
