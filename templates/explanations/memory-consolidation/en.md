Memory consolidation is a background process that passively runs whenever the agent has new experiences to process. Rather than having the agent curate its memory as it acts, which would dilute focus on both its task and its self-reflection, this process is trusted to amend its memory.

During memory consolidation, its new interactions are analyzed and filtered into its episodic and semantic memories, and the contents of its episodic memory is periodically revised to graduate information into longer or shorter term memory.

The episodic memory consists of a number of memory layers, each representing a different horizon - a short term layer may represent experiences retained for one day, while a long term layer may represent experiences that remain pertinent for a full year or longer. This way both core and contextual information is proportionally represented

Each of these layers condenses its experiences over this interval, into a retrospective, current perspective, and future perspective, grounding the AI in an ongoing narrative.

The semantic memory is a mesh of factual data, sorted into ontological categories. Persons, locations, events and so on, are isolated during memory consolidation, and added or updated to this ontological map, reflecting anything learned during the AI's interactions - such as Person->phone_number, or Event->Participants[]. The semantic memory surfaces these properties of relevant "entities," when the framework determines they are contextually relevant, or when the agent queries its memory.
